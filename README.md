### **최종 선정 결론**

GPT-5.4는 정확성, 한국어 자연스러움, 형식 준수 측면에서 가장 균형 잡힌 성능을 보여 최종 선정 모델로 판단된다.

특히 사실 기반 답변, 계산 안정성, 그리고 표·템플릿 같은 정해진 출력 형식을 잘 지킨다는 점에서 실무 활용성이 높다.

또한 한국어 표현이 자연스럽고 문맥 이해력이 안정적이어서, 일반 사용자 응대부터 보고서 작성, 분석 업무까지 폭넓게 적용할 수 있다.

비용이나 속도만 보면 다른 모델이 일부 강점을 가질 수 있지만, 전체적인 품질과 일관성을 종합하면 GPT-5.4가 가장 최적의 선택이다.

- AI 업무 과업 정의서
    
    #### **1. 업무 과업 (Task)**
    
    제공된 주간 회의 원문을 분석하여 핵심 의사결정 사항을 요약하고, 담당자별 실행 과제(Action Items)를 추출하여 체계적인 업무 보고서 형태로 가공합니다.
    
    #### **2. 원문 (Input Data - 예시)**
    
    *(실제 사용 시 이 자리에 회의 녹취록을 입력하세요)*
    
    > "김팀장: 오늘 회의 시작하죠. 지난번 마케팅 캠페인 성과가 좀 안 나왔는데, 이건 어떻게 할까요? 박대리님이 좀 파악해 보세요. 금요일까지 보고 부탁합니다. 그리고 이 대리, 이번 프로젝트 디자인 시안은 다 나왔나요?
    이대리: 아, 네. 거의 다 됐는데 로고 색상만 좀 변경하면 될 것 같습니다. 내일 오전까지 수정하겠습니다.
    김팀장: 좋습니다. 그리고 다음 달 워크샵 장소는 아직 미정이죠? 최 사원, 이번 주 내로 후보지 3곳 조사해서 공유해주세요.
    박대리: 아, 마케팅 건 말인데, 예산이 좀 부족해서 예비비 요청이 필요할 수도 있습니다.
    김팀장: 그래요? 그럼 그건 제가 따로 검토해보죠. 자, 여기까지 합시다."
    > 
    
    #### **3. 원하는 결과 (Output Format)**
    
    아래의 구조를 엄격히 준수하여 출력할 것.
    
    1. **[회의 개요]** 날짜, 참석자
    2. **[핵심 요약]** 3줄 내외의 요약
    3. **[의사결정 사항]** 결정된 이슈 및 배경
    4. **[Action Items]** (표 형식: 담당자 | 업무 내용 | 마감 기한)
    5. **[미해결 과제]** 후속 논의가 필요한 사항
    
    #### **4. 톤 (Tone)**
    
    - 사무적이고 객관적인 비즈니스 문체를 유지할 것.
    - 불필요한 감정적 수식어(예: "정말 훌륭한 회의였습니다", "열심히 하세요" 등)를 배제할 것.
    - 간결하고 명확한 문장으로 서술할 것.
    
    #### **5. 금지 (Prohibitions)**
    
    - **환각 금지:** 원문에 언급되지 않은 내용이나 사실을 임의로 추가하거나 지어내지 말 것.
    - **주관적 해석 금지:** 대화 내용에 대해 AI가 개인적인 의견이나 평가를 덧붙이지 말 것.
    - **불필요한 서론/결론 금지:** "회의록 요약입니다.", "도움이 되셨길 바랍니다."와 같은 관용적 인사를 생략하고 바로 본론부터 출력할 것.
    
    #### **6. 확인 질문 규칙 (Verification Rule)**
    
    - 원문 내용만으로 Action Items의 마감 기한이나 담당자가 명확하지 않을 경우, **임의로 추측하지 말 것.**
    - 대신, [Action Items] 표의 해당 칸에 **"기한 미언급"** 또는 "담당자 불분명"이라고 표기하고, 결과 하단에 "확인 필요 사항: [누락된 정보에 대한 질문]"을 별도로 명시하여 사용자에게 보완을 요청할 것.
